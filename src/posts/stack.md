### 一、栈 - 推荐题目

栈的核心思想是 **“后进先出”** 。做题时，要思考如何利用这个特性来解决问题，比如处理对称性、嵌套结构、回退操作等。

#### 🟢 入门级 (理解基本操作)

1.  **[20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/)**
    *   **考察点**：栈最经典的入门题。检验括号的对称性和嵌套顺序。
    *   **思路**：遇到左括号就压栈；遇到右括号就检查栈顶是否匹配的左括号，是则弹出，否则无效。最后栈必须为空。

2.  **[155. 最小栈](https://leetcode.cn/problems/min-stack/)**
    *   **考察点**：如何在常数时间内获取栈的最小值。考察对栈结构的扩展设计。
    *   **思路**：使用两个栈，一个正常存储数据，另一个同步存储当前的最小值。

#### 🟡 进阶级 (掌握核心应用)

3.  **[946. 验证栈序列](https://leetcode.cn/problems/validate-stack-sequences/)**
    *   **考察点**：模拟栈的入栈和出栈过程。理解栈操作序列的可能性。
    *   **思路**：用一个辅助栈模拟入栈 `pushed` 和出栈 `popped` 的过程。每次入栈后，循环判断栈顶是否等于 `popped` 序列的当前元素，是则弹出。

4.  **[232. 用栈实现队列](https://leetcode.cn/problems/implement-queue-using-stacks/)**
    *   **考察点**：理解栈和队列的区别，以及如何用 LIFO 实现 FIFO。
    *   **思路**：使用两个栈，一个 `inStack` 负责入队，一个 `outStack` 负责出队。当 `outStack` 为空时，将 `inStack` 的所有元素弹出并压入 `outStack`。

5.  **[1047. 删除字符串中的所有相邻重复项](https://leetcode.cn/problems/remove-all-adjacent-duplicates-in-string/)**
    *   **考察点**：栈在“消消乐”类问题中的应用。
    *   **思路**：遍历字符串，如果当前字符和栈顶字符相同，就弹出栈顶；否则压入栈。最后栈中剩下的就是结果。

#### 🔴 挑战级 (综合运用)

6.  **[739. 每日温度](https://leetcode.cn/problems/daily-temperatures/)**
    *   **考察点**：**单调栈** 的经典应用。单调栈是栈的高频考点，用于解决“下一个更大/更小元素”问题。
    *   **思路**：维护一个存储下标的栈，栈内温度对应的下标对应的温度是单调递减的。遍历温度数组，若当前温度高于栈顶下标对应的温度，则找到了栈顶元素的下一个更高温度，计算天数差并弹出栈顶。

7.  **[84. 柱状图中最大的矩形](https://leetcode.cn/problems/largest-rectangle-in-histogram/)**
    *   **考察点**：单调栈的困难题，理解如何确定每个柱子的左右边界。
    *   **思路**：维护一个单调递增栈。当当前柱子高度小于栈顶时，说明找到了栈顶柱子的右边界，可以计算以栈顶柱子为高的最大矩形面积。

### 二、队列 - 推荐题目

队列的核心思想是 **“先进先出”** 。常用于BFS（广度优先搜索）、滑动窗口等问题。

#### 🟢 入门级 (理解基本操作)

1.  **[225. 用队列实现栈](https://leetcode.cn/problems/implement-stack-using-queues/)**
    *   **考察点**：理解队列和栈的区别，以及如何用 FIFO 实现 LIFO。
    *   **思路**：使用一个队列。在压栈时，将新元素入队，然后将队列中原有的所有元素依次出队再入队，这样新元素就在队头了。

#### 🟡 进阶级 (掌握核心应用)

2.  **[933. 最近的请求次数](https://leetcode.cn/problems/number-of-recent-calls/)**
    *   **考察点**：队列在时间窗口统计问题中的应用。
    *   **思路**：将请求时间入队。每次新的请求进来，将队首所有不在 `[t-3000, t]` 范围内的请求出队，然后返回队列的大小即可。

3.  **[622. 设计循环队列](https://leetcode.cn/problems/design-circular-queue/)**
    *   **考察点**：对队列底层实现的深入理解，特别是如何利用数组和指针实现循环。
    *   **思路**：使用一个数组和两个指针 `front` 和 `rear`。关键点在于判空 (`front == rear`)、判满 (`(rear + 1) % capacity == front`) 和指针的移动。

#### 🔴 挑战级 (综合运用 - 常与BFS结合)

队列是 **广度优先搜索（BFS）** 算法的核心数据结构。以下题目虽然标签可能是二叉树或图，但本质是队列的应用。

4.  **[102. 二叉树的层序遍历](https://leetcode.cn/problems/binary-tree-level-order-traversal/)**
    *   **考察点**：队列在BFS中的标准用法。
    *   **思路**：将根节点入队。每次循环处理当前层的所有节点（通过当前队列大小控制），将它们的值存入结果，并将它们的子节点入队。

5.  **[542. 01 矩阵](https://leetcode.cn/problems/01-matrix/)**
    *   **考察点**：多源BFS，队列的经典应用。
    *   **思路**：将所有0的位置入队（作为多源起点），然后开始BFS，向四周扩散，更新每个1的位置到最近0的距离。

### 刷题建议

1.  **顺序**：严格按照从易到难的顺序刷，先栈后队列。
2.  **理解重于AC**：不要满足于通过。每做一题，都要理解为什么用栈/队列，它的“后进先出”/“先进先出”特性是如何在题目中体现的。
3.  **画图**：对于复杂的题目（如单调栈、循环队列），一定要在纸上画图模拟整个过程，这对理解非常有帮助。
4.  **总结模板**：比如单调栈、BFS层序遍历，都有相对固定的代码模板，多做几题后要自己总结出来。
5.  **反复练习**：栈和队列是基础，它们的应用会贯穿整个算法学习过程。现在打下的基础越牢固，后面学习递归、DFS、BFS、动态规划等就会越轻松。
